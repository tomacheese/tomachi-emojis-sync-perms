# AGENTS.md

## 目的
エージェント共通の作業方針を定義し、プロジェクトの整合性を維持する。

## 基本方針
- 会話と言語: 日本語を使用する。
- コメント言語: 日本語を使用する。
- エラーメッセージ言語: 英語を使用する。
- コミット規約: [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) に従い、`<description>` は日本語で記載する。
- 日本語と英数字の間: 半角スペースを挿入する。

## 判断記録のルール
- 判断内容の要約を記載する。
- 検討した代替案を列挙する。
- 採用しなかった案とその理由を明記する。
- 前提条件、不確実性を明示する。

## 開発手順（概要）
1. プロジェクトを深く理解し、既存のコードスタイルや規約を確認する。
2. `pnpm install` で最新の依存関係をインストールする。
3. 要件に基づいた変更を実装する。
4. `pnpm test` および `pnpm lint` で品質を検証する。

## セキュリティ / 機密情報
- Discord トークンや API キーなどの認証情報を Git にコミットしない。
- ログに機密情報を出力しないよう注意する。
- `.env` や `data/config.json` などの設定ファイルを適切に扱う。

## リポジトリ固有
- このプロジェクトは `discord.js` を使用したボットアプリケーションである。
- ロールの同期設定は YAML ファイルで管理されており、その構造を維持すること。
- `@book000/node-utils` ライブラリの規約に従うこと。
